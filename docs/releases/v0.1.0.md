# v0.1.0

Two new backends (**AArch64** and **WebAssembly**), a full **`-O2` optimization
tier** with an LLVM-class auto-vectorizer, **language threading** + a
**parallel-by-default compiler**, **incremental compilation**, and a **−74% cut**
in self-host compile RAM. 137 commits since v0.0.17.

This is the first release on the `0.1.x` line. The compiler now emits **five**
targets — Linux x86_64, Windows x86_64, Linux **AArch64**, **WebAssembly**
(wasm32/wasm64 + WASI), and Caustic's own CSE — and self-hosts to a byte-identical
fixpoint at every optimization level (`-O0`/`-O1`/`-O2`) on x86, and *on wasm*.

## New backends

- **WebAssembly** (`--target=wasm32-wasi` / `wasm64-wasi`) — emits a complete
  `.wasm` binary module directly, with no assembler/linker step. Every vreg is an
  i64 local; address-taken locals live on a shadow stack in linear memory;
  structured control flow via a dispatch loop (`br_table` on a `$state` local);
  positional calling convention; one backend, two memory models keyed on
  `target.arch`. Supports variadics, function pointers (funcref table +
  `call_indirect`), atomics (single-threaded lowering), the WASI
  filesystem/heap/argv, custom host imports (any non-WASI `extern "<module>"`
  becomes a host import for wasm↔JS interop), and `-O0`/`-O1`/`-O2`. **The
  compiler compiles ITSELF to a byte-identical wasm compiler** (gen1==gen2==gen3)
  that compiles itself again — runs under the standard `node:wasi` and Deno.
  *Experimental.*
- **Linux AArch64** (`--target=linux-aarch64`) — static AArch64 ELF via a scalar
  stack-home backend (no regalloc), the all-spill model the wasm backend mirrors.
  Atomics and threads supported. *Experimental.*

## Optimization: a real `-O2` tier

- **Global value numbering** (cross-block CSE) over the existing CFG + dominator
  tree, full loop unrolling for constant-trip loops, and GVN extended to float ops.
- **SIMD auto-vectorizer** — reductions (i64/i32/f64), elementwise maps
  (`c[i]=a[i] OP b[i]`, `+ − × ÷`) and broadcasts (`c[i]=a[i] OP k`), over constant
  *and* runtime lengths, with runtime alias multiversioning and a cpuid-dispatched
  **AVX2** (256-bit) path falling back to SSE2 then a scalar remainder. AVX2 is
  emitted as raw VEX bytes — no assembler change.
- **vec2** — the vectorizer was rewritten from a pattern-matcher into an LLVM-class
  Legal→Plan→Execute loop vectorizer (natural-loop discovery, mini-SCEV
  affine-address analysis, IV/recurrence descriptors, an LAA-shaped
  memory-dependence test + legality gate); v1 was retired.
- **`-O2` GVN miscompile fix** — GVN was CSEing comparison results across blocks,
  which is unsound against the instruction selector's compare-into-branch fusion:
  when a comparison feeds a conditional branch, isel emits `cmp; jcc` and never
  materialises the boolean, so a CSE'd second consumer read a slot the fused branch
  never wrote → garbage. This silently broke the `-O2` self-host (a compiler built
  at `-O2` produced wrong code) while `-O0`/`-O1` stayed fine. Restricting GVN to
  arithmetic/address ops fixes it — and is why the new pre-commit gate tests all
  three optimization levels.

The x86 auto-vectorizer stays x86-only (its runtime AVX2 dispatch + 256-bit ops
don't map to wasm's 128-bit v128); `-O2` wasm is correct scalar code the engine
vectorizes itself.

## Threading + a parallel compiler

- **Language threading** — atomic builtins (`atomic_load/store/add/xchg/cas` +
  `fence()`, seq_cst), `with naked` functions, `std/thread.cst` (1:1 OS threads via
  raw `clone(2)` on Linux / `CreateThread` on Windows), `std/sync.cst` (Spinlock,
  Mutex, Cond, RwLock, Once, WaitGroup, bounded MPSC Channel over a futex shim), and
  thread-safe allocators (lazy futex lock, no-op single-thread fast path).
- **Parallel-by-default builds** — `caustic x.cst -o out` forks per-module emission
  across all cores (dynamic work-stealing over a `MAP_SHARED` task counter); the
  whole-program inline pass runs per-worker off a pristine snapshot (cross-module
  inlining preserved) and the frontend optimize is deferred to the workers.
  Byte-identical across `-j`. ~2.1× on the self-host.
- **Dynamic stack growth** — a recursive / indirect-call program no longer needs
  `--stack-size`: a prologue stack probe grows the committed stack on demand.

## Incremental compilation

Separate compilation with per-module `.csti` binary interfaces (types / signatures
/ imut-bodies / generic templates; regular bodies stripped), topological link-time
DCE, and a `--module-objects` batch cold-build that runs the whole-program frontend
once and loops codegen+assemble per module. Clean build of the compiler
**4.8s → 1.37s (~3.5×)**, dev-loop ~247ms.

## Compile-time RAM −74%, I/O −78%

- Self-host `-O1` peak RSS **360 MB → ~92 MB (−74%)**: bump-allocate the transient
  body-AST at exact size (no slab class rounding) and free it right after IR-gen,
  route generic-instance clones through the same transient arena, narrow
  low-cardinality `Node`/`IRInst`/`Variable` fields, move rare decl-only `Node`
  fields to a lazy side-struct, free imported-module token arrays after semantic,
  and grow the body-node arena on demand (VmPeak 430 → 201 MB).
- **Regalloc O(n²) → O(degree)**: the `-O1` graph-coloring allocator scanned all n
  vregs to enumerate a node's neighbors; an adjacency-list index (bit matrix kept as
  the O(1) pair-query oracle) plus dense vreg compaction cut per-gen build from
  1.27s to 0.97s.
- **In-memory `.s` pipeline** (codegen → peephole → assembler, no disk round-trip)
  + `--no-asm-cache`: self-host compile I/O ~32 MB → ~7 MB (−78%).

## Correctness fixes (root-cause)

- **Octal escapes `\NNN`** were silently corrupted: the string decoder only handled
  `\0`, so `\012`/`\377`/`\200` dropped the backslash (every byte ≥ 128 unreachable
  via octal). Both string and char paths now decode C-style 1–3 digit octal. A
  follow-up fixed the `.string` emitter fusing a NUL byte with a following
  octal-digit byte (`\0` + `'5'` → `\05`), dropping a byte.
- **Register-allocation clobber points** — `SET_CTX` (`xor r10,r10`) and struct copy
  (`rep movsb`, clobbers rdi/rsi/rcx) are now recorded as clobber points so a value
  live across them is evicted from the clobbered caller-saved register; fixes `-O1`
  miscompiles in closure/method-call and struct-field-copy code.

## Tooling

- **Native self-host pre-commit gate** (`tools/precommit.sh`) — builds the toolchain
  from source and self-hosts to a 4-generation fixpoint at `-O0`, `-O1` **and**
  `-O2` (gen2==gen3==gen4 byte-identical + gen4 must compile correctly), checks
  every example is identical across opt levels, and runs the unit suite.
- **Release scripts** — `tools/prerelease.sh` refuses to release without a version
  bump; `tools/release-build.sh` builds the shipped toolchain as the settled
  **gen4 `-O2`** self-host fixpoint (4-phase sequence: compiler → gen4, toolchain →
  gen4, compiler → gen4 again) before packaging.
- **LSP** — `handlers.cst` split into nav/complete/sig modules; hover docs +
  completions for the atomics and target builtins.

## Bootstrap

The shipped binary is the **generation-4 `-O2`** self-host fixpoint. Linux x86_64
ELF, `gen2 == gen3 == gen4` byte-identical at each level:

| Level | gen2 == gen3 == gen4 (sha256) |
|-------|-------------------------------|
| `-O0` | `42d73f9d3d7eb8870e50fb53aaa3912d4f5a4d09bdb85d519e40cb6e53100d51` |
| `-O1` | `6b3b9a2e2da571f6c702e2d86d442bb0ff797fbc4f3fee9c4cfd92868111f2e7` |

The compiler also self-hosts to a byte-identical wasm compiler under `node:wasi`
(gen1 == gen2 == gen3).

## Install (Linux)

```bash
tar xzf caustic-x86_64-linux.tar.gz
sudo cp caustic-x86_64-linux/bin/* /usr/local/bin/
sudo mkdir -p /usr/local/lib/caustic
sudo cp -r caustic-x86_64-linux/lib/caustic/std /usr/local/lib/caustic/
```

Or the one-liner:

```bash
curl -fsSL https://raw.githubusercontent.com/Caua726/Caustic/main/install.sh | sh
```

# Supported x86_64 Instructions

caustic-as supports a focused subset of the x86_64 instruction set, covering all instructions generated by the Caustic compiler's codegen phase. Each instruction is identified by a constant `INST_*` value defined in `encoder.cst`.

## Data Movement

| Mnemonic | ID | Operand Forms | Description |
|----------|----|---------------|-------------|
| `mov` | 1 | reg,reg / reg,imm / reg,[mem] / [mem],reg / [mem],imm / reg,[rip+label] | Move data. Supports 8/16/32/64-bit operands. 64-bit immediate loads use `movabs` encoding when value exceeds 32 bits. |
| `movsx` | 2 | reg,reg / reg,[mem] | Move with sign extension (byte/word to larger). Uses `0F BE` (byte) or `0F BF` (word). |
| `movsxd` | 3 | reg,reg / reg,[mem] | Move dword to qword with sign extension. Opcode `63` with REX.W. |
| `movzx` | 4 | reg,reg / reg,[mem] | Move with zero extension. Uses `0F B6` (byte) or `0F B7` (word). |
| `lea` | 5 | reg,[mem] / reg,[rip+label] | Load effective address. Always 64-bit (REX.W). |
| `movq` | 53 | xmm,reg / reg,xmm | Move quadword between GPR and XMM register. `66 REX.W 0F 6E` (to xmm), `66 REX.W 0F 7E` (from xmm). |

## Arithmetic

| Mnemonic | ID | Operand Forms | Description |
|----------|----|---------------|-------------|
| `add` | 6 | reg,reg / reg,imm / reg,[mem] | Integer addition. Supports imm8 sign-extended form. |
| `sub` | 7 | reg,reg / reg,imm / reg,[mem] | Integer subtraction. |
| `imul` | 8 | reg,reg | Signed multiply (two-operand form). Opcode `0F AF`. |
| `idiv` | 9 | reg | Signed divide rdx:rax by operand. Opcode `F7 /7`. |
| `div` | 62 | reg | Unsigned divide rdx:rax by operand. Opcode `F7 /6`. |
| `neg` | 11 | reg | Two's complement negate. Opcode `F7 /3`. |
| `cqo` | 10 | (none) | Sign-extend rax into rdx:rax (64-bit). `REX.W 99`. |
| `cdq` | 61 | (none) | Sign-extend eax into edx:eax (32-bit). `99`. |

## Bitwise

| Mnemonic | ID | Operand Forms | Description |
|----------|----|---------------|-------------|
| `and` | 12 | reg,reg / reg,imm / reg,[mem] | Bitwise AND. |
| `or` | 13 | reg,reg / reg,imm / reg,[mem] | Bitwise OR. |
| `xor` | 14 | reg,reg / reg,imm / reg,[mem] | Bitwise XOR. Common idiom: `xor reg, reg` to zero a register. |
| `not` | 15 | reg | Bitwise NOT (one's complement). Opcode `F7 /2`. |
| `shl` | 16 | reg,cl / reg,imm | Shift left. Uses `D3 /4` (by cl) or `C1 /4 imm8` (by immediate). Special case: shift by 1 uses opcode `D1`. |
| `shr` | 17 | reg,cl / reg,imm | Logical shift right. `/5` extension. |
| `sar` | 18 | reg,cl / reg,imm | Arithmetic shift right (preserves sign). `/7` extension. |

## Comparison and Testing

| Mnemonic | ID | Operand Forms | Description |
|----------|----|---------------|-------------|
| `cmp` | 19 | reg,reg / reg,imm / reg,[mem] | Compare (subtract without storing result, sets flags). |
| `test` | 20 | reg,reg | Bitwise AND without storing result, sets flags. |

## Conditional Set (SETcc)

All SETcc instructions take a single 8-bit register operand and set it to 1 or 0 based on flags.

| Mnemonic | ID | Condition |
|----------|----|-----------|
| `sete` | 21 | Equal (ZF=1) |
| `setne` | 22 | Not equal (ZF=0) |
| `setl` | 23 | Less (signed) |
| `setle` | 24 | Less or equal (signed) |
| `setg` | 25 | Greater (signed) |
| `setge` | 26 | Greater or equal (signed) |
| `setb` | 27 | Below (unsigned) |
| `setbe` | 28 | Below or equal (unsigned) |
| `seta` | 29 | Above (unsigned) |
| `setae` | 30 | Above or equal (unsigned) |

Encoding: `0F 9x /0 ModR/M(11, 0, reg)`. REX prefix emitted when the register is r8b-r15b or spl/bpl/sil/dil.

## Stack Operations

| Mnemonic | ID | Operand Forms | Description |
|----------|----|---------------|-------------|
| `push` | 31 | reg | Push register onto stack. Opcode `50+rd`. REX.B for extended registers. |
| `pop` | 32 | reg | Pop from stack into register. Opcode `58+rd`. |

## Control Flow

| Mnemonic | ID | Operand Forms | Description |
|----------|----|---------------|-------------|
| `call` | 33 | label | Call function. Opcode `E8` + rel32. |
| `ret` | 34 | (none) | Return from function. Opcode `C3`. |
| `jmp` | 35 | label | Unconditional jump. Opcode `E9` + rel32. |

## Conditional Jumps (Jcc)

All conditional jumps use the near form with 32-bit relative offset: `0F 8x rel32` (6 bytes total).

| Mnemonic | ID | Condition Code | Description |
|----------|----|----------------|-------------|
| `jz` | 36 | 0x84 | Jump if zero |
| `jnz` | 37 | 0x85 | Jump if not zero |
| `je` | 38 | 0x84 | Jump if equal (same as jz) |
| `jne` | 39 | 0x85 | Jump if not equal (same as jnz) |
| `jl` | 40 | 0x8C | Jump if less (signed) |
| `jle` | 41 | 0x8E | Jump if less or equal (signed) |
| `jg` | 42 | 0x8F | Jump if greater (signed) |
| `jge` | 43 | 0x8D | Jump if greater or equal (signed) |
| `ja` | 44 | 0x87 | Jump if above (unsigned) |
| `jae` | 45 | 0x83 | Jump if above or equal (unsigned) |
| `jb` | 46 | 0x82 | Jump if below (unsigned) |
| `jbe` | 47 | 0x86 | Jump if below or equal (unsigned) |

## SSE (Scalar Double-Precision)

All SSE instructions use the `F2` prefix (scalar double) and operate on XMM registers.

| Mnemonic | ID | Operand Forms | Description |
|----------|----|---------------|-------------|
| `addsd` | 54 | xmm,xmm | Add scalar double. `F2 0F 58`. |
| `subsd` | 55 | xmm,xmm | Subtract scalar double. `F2 0F 5C`. |
| `mulsd` | 56 | xmm,xmm | Multiply scalar double. `F2 0F 59`. |
| `divsd` | 57 | xmm,xmm | Divide scalar double. `F2 0F 5E`. |
| `xorpd` | 58 | xmm,xmm | Bitwise XOR packed double. `66 0F 57`. Used for zeroing XMM registers. |
| `cvttsd2si` | 59 | reg,xmm | Convert double to integer (truncation). `F2 REX.W 0F 2C`. |
| `cvtsi2sd` | 60 | xmm,reg | Convert integer to double. `F2 REX.W 0F 2A`. |

## Miscellaneous

| Mnemonic | ID | Operand Forms | Description |
|----------|----|---------------|-------------|
| `syscall` | 48 | (none) | System call. Opcode `0F 05`. |
| `cld` | 49 | (none) | Clear direction flag. Opcode `FC`. |
| `rep` | 50 | (none) | Repeat prefix. Opcode `F3`. Emitted as a separate instruction; the following `movsb` is processed on the next iteration. |
| `movsb` | 51 | (none) | Move byte string (used with `rep`). Opcode `A4`. |
| `nop` | 52 | (none) | No operation. Opcode `90`. |

## Instruction Size Variation

x86_64 instructions have variable length. The same logical instruction can be different sizes depending on operand types and values:

- `mov rax, 0` = 7 bytes (REX.W + C7 /0 + ModR/M + imm32, using sign-extended form)
- `mov rax, 0x100000000` = 10 bytes (REX.W + B8+rd + imm64, value exceeds 32 bits)
- `add rax, 1` = 4 bytes (REX.W + 83 /0 + ModR/M + imm8, sign-extended)
- `add rax, 256` = 7 bytes (REX.W + 81 /0 + ModR/M + imm32, value exceeds 8 bits)
- `mov [rbp - 8], rax` = 4 bytes (REX.W + 89 + ModR/M(01) + disp8)
- `mov [rbp - 256], rax` = 7 bytes (REX.W + 89 + ModR/M(10) + disp32)

This variable sizing is why pass 1 must trial-encode each instruction to determine its exact byte count before computing label offsets.
